## 🚀 Technical Features

* **State-Driven UI:** Custom JavaScript logic to toggle between the signup form and the success state dynamically based on user interaction.
* **Real-Time Validation:** Email format checking with immediate visual error feedback and accessible messaging.
* **Modern Layout Coordination:** Use of **CSS Grid** and **Flexbox** to handle the asymmetrical desktop design and stacked mobile view.
* **Responsive Art Direction:** Strategic asset management where images adapt and scale to maintain visual impact across devices.
* **Apple-Style Minimalist UI:** Focus on clean typography, precise spacing, and subtle shadows for a premium, professional feel.

## 🛠️ Tech Stack

- **Structure:** HTML5 (Semantic & Accessible)
- **Styling:** CSS3 (Custom Properties, Grid, Flexbox)
- **Logic:** Vanilla JavaScript (ES6+)
- **Deployment:** Vercel

## 🧠 Key Learnings

Building this component was an exercise in **state management** and **UX feedback loops**.

**Main Takeaways:**
- Mastering the transition between different **UI views** without page reloads using JavaScript.
- Implementing logic to ensure data integrity and provide real-time validation for email inputs.
- Organizing CSS variables to manage a consistent professional color palette throughout the project.
- Developing accessible form patterns that guide the user clearly through error and success states.

2.  **Navigate to the folder:**
